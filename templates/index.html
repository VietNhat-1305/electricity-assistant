🧮 Tối ưu DP vs Greedy

So sánh thuật toán Quy hoạch động Group Knapsack và Greedy

Giải thích thuật toán
📌 Bài toán Group Knapsack
1
Input
n thiết bị, mỗi cái có công suất (W), độ ưu tiên, giờ tối đa/ngày. Ngân sách W kWh.
2
Lựa chọn
Với mỗi thiết bị i, chọn h ∈ {0, 0.5, 1.0, …, max_h}:
cost(i,h) = power_w × h / 1000 (kWh) val(i,h) = priority × h (điểm)
3
Mục tiêu
Tối đa hóa Σ val(i, h_i) sao cho Σ cost(i, h_i) ≤ W
⚙️ Chuyển trạng thái DP
dp[i+1][j] = max( dp[i][j], ← không dùng dp[i][j - cost_u] + val ← dùng h giờ ) choice[i][j] = số giờ đã chọn
📊 Độ phức tạp
DP: O(n × W/δ × H/Δ) — tối ưu toàn cục Greedy: O(n log n) — xấp xỉ, nhanh hơn δ = 0.01 kWh | Δ = 0.5h
